Can CLI watch serialize the Rust-reachable session?

Question: can CLI watch serialize the final Rust-reachable Pack-bound session lifecycle without inventing adapter state?

Answer under test

Yes, if the boundary stays explicit.

The Rust session owns lifecycle, revisions, evaluations, attempts, fences, generations, publication, and last success. CLI separately owns the Pack-to-session binding, Current Delivery, and Last Successful Delivery. A different Pack ends the old session and creates another session outside the reducer.

Assumption

The regenerated fixture is the final reachable Rust interface. These are immutable expected traces, not an implementation of CompilationSession::apply.
