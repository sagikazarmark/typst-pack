## Project roots and path resolution

### Language and compiler model

Typst path strings are virtual and forward-slash based. A relative string resolves from the parent of the Typst file containing the operation; a leading `/` resets resolution to that file's root. `.` is removed, `..` removes one virtual segment, repeated separators are normalized, and a parent traversal at `/` is rejected. ([0.15.0 path documentation](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/foundations/path.rs#L9-L72), [0.15.0 resolver](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/foundations/path.rs#L177-L250), [current resolver](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-library/src/foundations/path.rs#L177-L250))

`VirtualRoot` distinguishes the project from `Package(PackageSpec)`. Relative and absolute paths retain the root of the referring file, so an absolute path inside package code points to that package's root, not the project. A constructed `path` value retains its resolved root and can therefore be explicitly passed across file or package boundaries. ([rooted identity](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L16-L81), [package-root documentation](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/foundations/path.rs#L84-L127), [current rooted identity](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-syntax/src/path.rs#L16-L81))

The documented project root is the main file's parent by default; the CLI can select a different ancestor, and the web app treats the whole web project as the root. The main file must be in the selected root's subtree. ([0.15.0 project-root documentation](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/foundations/path.rs#L84-L109))

### CLI host mapping

For a path input, `typst-cli` canonicalizes the input, chooses `--root`/`TYPST_ROOT` or the canonical input's parent, canonicalizes that root, and then virtualizes the input by stripping the root prefix. This makes an input symlink's target, rather than its lexical link location, determine whether the main file is inside the root. Without a path input or explicit root, the fallback root is the canonical current directory. ([0.15.0 arguments](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/args.rs#L396-L430), [0.15.0 world construction](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L193-L243), [current world construction](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-cli/src/world.rs#L193-L243))

The compiler itself receives no host project-root path. Its `World` receives a main `FileId` and must answer source/file requests for other `FileId`s; the CLI's canonical host root exists only in `SystemWorld` and its `FileLoader`. ([0.15.0 `World`](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/lib.rs#L44-L98), [0.15.0 `SystemWorld`](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L24-L145), [current `World`](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-library/src/lib.rs#L44-L98))

## Source and raw-file identity

`FileId` is the interned form of a `RootedPath`, which is `(VirtualRoot, VirtualPath)`. Re-interning an equal rooted virtual path returns the same ID; `FileId::unique` deliberately creates a distinct ID for virtual inputs such as stdin even when the attached rooted path is equal. ([0.15.0 interner](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L83-L173), [stdin identity](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L167-L183))

`VirtualPath` identity is a normalized Unicode segment string. It does not consult the filesystem when virtualizing or realizing and does not canonicalize case, symlinks, hard links, or inode identity. Consequently, two virtual aliases to the same host object remain distinct `FileId`s, while the same normalized rooted path is one identity. ([0.15.0 virtual path](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L189-L249), [normalization](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L401-L449), [current virtual path](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-syntax/src/path.rs#L189-L249))

The `World` contract uses the same `FileId` for `source(id)` and `file(id)`, and requires `file(id)` to succeed whenever `source(id)` succeeds. `typst-kit::FileStore` implements both views in one per-ID slot. A source must decode as UTF-8 and has a UTF-8 BOM removed; the raw file retains the original bytes, including a BOM. ([0.15.0 `World` file contract](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/lib.rs#L69-L88), [0.15.0 file slots](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L128-L237), [current file slots](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-kit/src/files.rs#L128-L237))

Imports/includes request `source(FileId)`. Data loaders such as `read`, images, CSV, JSON, YAML, and bibliographies resolve the same rooted identity and request `file(FileId)`. Package import additionally requests the package's `typst.toml` as raw bytes before requesting its entrypoint as source. ([0.15.0 import loading](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-eval/src/import.rs#L211-L283), [0.15.0 data-source loading](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/loading/mod.rs#L46-L110))

## Root escape and symlink behavior

Typst rejects lexical parent traversal above the virtual root and rejects backslashes. In 0.15.0 it also validates each virtual segment before joining it to a host root, including rejecting Windows drive-like components and reserved device names; the 0.15.0 release history calls out those Windows fixes and the new rejection of non-Unicode input paths. ([0.15.0 normalization and errors](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L467-L522), [0.15.0 escape error](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L524-L628), [0.15.0 release history](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/docs/content/changelog/0.15.0.typ#L266-L280))

This is lexical confinement, not physical confinement. `VirtualPath::realize` explicitly performs no filesystem operation, and `FsRoot::load` joins the virtual path to the root and then calls ordinary metadata/read operations while noting that the result can still escape through symlinks. An in-root symlink to an outside file or directory is therefore followed by this implementation for both project and package roots. ([0.15.0 realization](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L229-L249), [0.15.0 `FsRoot`](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L324-L359), [current `FsRoot`](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-kit/src/files.rs#L324-L359))

There is a deliberate difference between main-input setup and later reads: the CLI canonicalizes the main input and project root once, but `FsRoot` does not canonicalize each requested file. Thus an input symlink targeting outside is rejected by root-prefix validation, while a symlink encountered below an accepted root can lead outside. ([0.15.0 setup](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L199-L241), [0.15.0 load path](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L246-L269))

## Package lookup

A source package reference is a namespace/name/version triple such as `@preview/example:0.1.0`. The compiler identifies every package file with `VirtualRoot::Package(PackageSpec)` plus a package-relative `VirtualPath`; it reads `typst.toml`, validates the manifest's name/version/compiler bound, resolves the declared entrypoint within that same package root, and imports it as source. ([language documentation](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/docs/content/reference/language/scripting.typ#L243-L254), [package identity](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/package.rs#L224-L265), [manifest resolution](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-eval/src/import.rs#L247-L283))

The CLI's `SystemPackages` lookup order is:

1. the package data directory;
2. the package cache directory;
3. a download from Typst Universe for the `preview` namespace, unpacked into the cache, followed by another cache lookup.

If no cache directory is configured/available, the implementation does not take the download branch. A non-`preview` package is local-only. ([0.15.0 lookup order](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/packages.rs#L20-L124), [Universe namespace](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/packages.rs#L309-L379), [current lookup order](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-kit/src/packages.rs#L20-L124))

`--package-path`/`TYPST_PACKAGE_PATH` replaces the default data root, and `--package-cache-path`/`TYPST_PACKAGE_CACHE_PATH` replaces the default cache root. Each filesystem package root is laid out as `namespace/name/version`; defaults are OS data/cache directories under `typst/packages`. These are single configured roots, not ordered CLI path lists. ([0.15.0 arguments](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/args.rs#L450-L464), [CLI construction](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/packages.rs#L1-L18), [filesystem package layout/defaults](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/packages.rs#L145-L195))

The official registry protocol and index are not stable interfaces: upstream explicitly says there is no standardized registry protocol and that the official index format is unspecified and may change at any time. ([registry implementation](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/packages.rs#L309-L335), [index stability](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/packages.rs#L414-L437))

## Font lookup

The compiler-facing interface contains no font paths. `World::book()` provides metadata for all known fonts, and `World::font(index)` returns bytes parsed as a font for an index from that book. The contract permits an index from an outdated/different book during incremental validation. ([0.15.0 `World`](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/lib.rs#L60-L97), [current `World`](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-library/src/lib.rs#L60-L97))

The default CLI builds its font store lazily by scanning system fonts, adding compiled-in fonts unless disabled, then recursively scanning each `--font-path`/`TYPST_FONT_PATHS` directory. The official CLI enables embedded and scan-font features; the embedded set is Libertinus Serif, New Computer Modern, New Computer Modern Math, and Deja Vu Sans Mono. System locations and discovered faces are delegated to `fontdb` and are platform-dependent. ([0.15.0 CLI discovery](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/fonts.rs#L36-L55), [font arguments](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/args.rs#L466-L490), [font providers](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/fonts.rs#L111-L194), [CLI features](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/Cargo.toml#L58-L71))

The exact 0.15/current code has a precedence ambiguity. The 0.7 release history says explicitly supplied font paths have priority over system and embedded fonts, but 0.15/current discovery inserts system, embedded, then explicit-path faces, while equal metadata scores keep the first inserted face. This establishes the observed implementation order but does not establish whether the changed tie outcome is intended. ([0.7 promise](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/docs/content/changelog/0.7.0.typ#L80-L85), [0.15.0 insertion order](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/fonts.rs#L36-L55), [0.15.0 tie selection](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/text/font/book.rs#L71-L163), [current insertion order](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-cli/src/fonts.rs#L36-L55))

Font files are loaded through `FontStore`, not `FileStore`: metadata is scanned up front, individual filesystem font bytes are loaded once on first use, and neither operation creates a `FileId`. This separation determines the dependency and watch omissions below. ([0.15.0 font store](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/fonts.rs#L15-L127), [current font store](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-kit/src/fonts.rs#L15-L127))

## Dependency observation

`World` has no dependency-list or observation method. `typst-kit::FileStore` adds an integration-level access log: every per-ID slot requested through `source` or `file` since the last reset is a dependency, the order is arbitrary, and reset clears accessed markers while retaining stale source objects for incremental reuse. ([0.15.0 `World`](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/lib.rs#L44-L98), [0.15.0 `FileStore` dependencies](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L19-L116), [current `FileStore` dependencies](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-kit/src/files.rs#L19-L116))

An attempted read counts even when loading returns `NotFound` or another error. The upstream tests explicitly expect a missing raw file in the dependency set. Compiler diagnostic probes can also add IDs: for example, an invalid non-`.typ` main source causes the compiler to probe the same path with a `.typ` extension to offer a hint. The set is therefore an attempted-access log, not merely successfully consumed content. ([missing-file test](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L397-L418), [diagnostic probe](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst/src/lib.rs#L206-L243))

`SystemWorld::dependencies` resolves accessed IDs back to host paths. Project and successfully obtained package files are included, with one entry per accessed `FileId`; a package ID that still cannot be resolved is silently dropped. Project paths are presented relative to the current directory when possible, while paths outside the project root, including package-cache/data files, remain host paths. ([0.15.0 world dependency resolution](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L87-L107), [root/package resolution](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L246-L269), [CLI path presentation](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/deps.rs#L164-L177))

The CLI writes dependency output after compilation/export and diagnostic printing. JSON and zero-delimited output are therefore attempted even after Typst compilation errors; Make output is emitted only when successful output paths are available. Dependency order is not stabilized by the CLI. ([compile flow](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/compile.rs#L254-L339), [dependency formats](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/deps.rs#L9-L125))

Fonts, font directories, package data/cache directory searches, registry/index/network requests, and package archive installation are not `FileStore` IDs and are absent. Once a package root exists, its requested `typst.toml`, source, and raw files are ordinary IDs and are present. ([package import requests](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-eval/src/import.rs#L257-L283), [file-only access log](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L65-L99), [separate font store](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/fonts.rs#L15-L127), [package acquisition](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/packages.rs#L84-L124))

## Watch invalidation

`typst watch` compiles once, then repeatedly replaces the watch set with the latest `SystemWorld::dependencies()`, waits, resets `FileStore`, recompiles, and asks `comemo` to evict old cache entries. A failed compilation still reaches this loop because `compile_once` prints diagnostics and returns normally for Typst source errors. ([0.15.0 watch loop](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/watch.rs#L17-L83), [0.15.0 compile error handling](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/compile.rs#L254-L314), [current watch loop](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-cli/src/watch.rs#L17-L83))

Existing dependencies are watched as individual non-recursive paths. Missing paths cannot be registered with `notify`, so the watcher polls them every 300 ms until one appears. The initial missing input/root is handled similarly before a `SystemWorld` exists. This makes a missing project/package file watchable when its root resolves, because failed file requests are dependencies. ([watch setup and missing roots](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/watch.rs#L26-L49), [watch-set update](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/watcher.rs#L35-L118), [current watch-set update](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-kit/src/watcher.rs#L35-L118))

Create, data-modify, rename, and remove events invalidate; access, metadata-only, and `Other` events do not. Events are batched for 100 ms, capped at 500 ms, and output-file-only events are ignored. The source notes that output suppression is incomplete for multi-file image export. ([0.15.0 event handling](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/watcher.rs#L120-L210), [current event handling](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-kit/src/watcher.rs#L120-L210))

Watch invalidation inherits the access log's boundaries:

- Used and missing project files, and files below an obtainable package root, participate. ([file access log](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L81-L116), [root resolution](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L246-L269))
- A wholly unavailable package is dropped when its `FileId` cannot be resolved to a host path, so creating/installing that package is not itself watched. ([dependency filter](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L97-L101), [package obtain failure](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/packages.rs#L95-L124))
- Font files and directories never participate. Paged watch eagerly scans the lazy font store once, and `FontStore` caches each loaded font independently of `FileStore::reset`. ([paged watch font scan](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/watch.rs#L51-L57), [font caching](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/fonts.rs#L63-L127))

## Stable interface versus adapter details

"Stable" here means the strongest boundary upstream actually exposes for the pinned compiler, not a promise that pre-1.0 Typst Rust APIs or language behavior never change. Typst's own release history records breaking path-language changes in 0.15.0 and an earlier breaking `World` redesign. ([0.15.0 breaking path changes](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/docs/content/changelog/0.15.0.typ#L28-L35), [earlier `World` break](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/docs/content/changelog/0.6.0.typ#L28-L46))

| Layer | What upstream establishes | What it does not establish |
| --- | --- | --- |
| Documented language behavior | Relative/absolute virtual path resolution, lexical root escape rejection, distinct project/package roots, and transferable rooted `path` values. ([path docs](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/foundations/path.rs#L31-L127)) | A host directory layout, symlink confinement algorithm, package cache, or font search policy. |
| Compiler embedding interface | `World::{main, source, file, book, font}` and `FileId` rooted virtual identity for exactly the embedded version. ([`World`](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/lib.rs#L44-L98), [`FileId`](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L16-L173)) | Filesystem access, package download, dependency reporting, watch, async I/O, or physical provenance. |
| `typst-kit` | Optional, feature-gated building blocks for files, fonts, packages, and watching; upstream calls it a source of truth for integrations, but all features default off. ([kit scope](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/lib.rs#L1-L30)) | A requirement that a custom `World` use these implementations or reproduce every CLI detail. |
| `typst-cli` | User-visible defaults and controls: canonical root selection, package/font environment variables and flags, dependency formats, and watch behavior. ([world args](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/args.rs#L396-L490), [dependency args](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/args.rs#L360-L376)) | A compiler-level requirement for non-CLI integrations. |
| Incidental implementation | Hash-map dependency order, exact polling/batching intervals, `notify` event filtering, lazy/once-only font caching, and host-path presentation. ([file store](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L81-L116), [watcher constants](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/watcher.rs#L35-L45), [dependency paths](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/deps.rs#L164-L177)) | Portability, deterministic ordering, or compatibility across compiler upgrades. |

## Architecture implications for the wayfinder map

These are constraints and seams exposed by upstream, not typst-pack policy choices.

1. A portable project model needs a logical identity equivalent in expressive power to `(project-or-package root, normalized virtual path)` and must not accidentally substitute a host path, canonical path, or archive offset for `FileId` identity. The same logical identity can be requested as source and raw bytes with different decoding behavior. ([identity](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L16-L98), [source/raw contract](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/lib.rs#L69-L88))
2. Host project-root selection and physical confinement belong outside the compiler core. Any design can preserve Typst's virtual path semantics while independently representing whether physical symlink escape is allowed, denied, or not applicable for an in-memory/archive provider. Upstream's own mapping separates `VirtualPath::realize` from `FsRoot::load`. ([realization seam](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-syntax/src/path.rs#L229-L249), [filesystem adapter](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L324-L359))
3. Compiler file access, package acquisition, and font provisioning are separate upstream channels. A dependency/provenance model that observes only `source`/`file` calls can match `typst-cli --deps` yet still omit selected font bytes and the package lookup/download inputs needed for reproducibility. ([`World` channels](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/lib.rs#L60-L97), [file-only observation](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L65-L99))
4. Observation at the I/O seam can record attempted accesses, including missing paths, but a portable closure representation also needs to distinguish logical identity from provider/physical provenance. The CLI erases that distinction when it converts every ID to a `PathBuf`, and it silently loses unresolved package identities. ([access log](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L81-L116), [CLI conversion](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L97-L101))
5. Watch is downstream of dependency and provenance design. Reproducing only upstream's latest-file-access loop also reproduces its omissions for fonts and unavailable packages; supporting non-filesystem providers additionally requires an invalidation mechanism that is not present in `typst-cli`'s `PathBuf` watcher. ([watch loop](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/watch.rs#L67-L83), [`PathBuf` watcher](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/watcher.rs#L19-L33))
6. CLI parity should be version-scoped. The current-upstream comparison found no semantic delta for this ticket, but upstream has changed path language, CLI dependencies, font discovery, and `World` in prior releases, so an engine upgrade still requires an explicit parity review rather than treating these findings as timeless. ([0.15.0 changes](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/docs/content/changelog/0.15.0.typ#L266-L284), [0.6.0 changes](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/docs/content/changelog/0.6.0.typ#L28-L46))

## Unresolved upstream facts

1. **Symlink security intent:** the user documentation says a project can access only paths within its root, while the path and `FsRoot` implementations explicitly acknowledge symlink escape. The first-party sources establish current behavior but do not say whether it is a supported semantic, accepted limitation, or future hardening target. ([documentation](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/foundations/path.rs#L84-L104), [implementation caveat](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L342-L359))
2. **Font precedence intent:** first-party release history promises explicit-path priority, but 0.15/current insertion and tie-selection code do not encode unconditional source priority. No material in the surveyed 0.15.0, 0.15.1, or current-main release/source history resolves whether this is intentional. ([release-history claim](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/docs/content/changelog/0.7.0.typ#L80-L85), [current discovery](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-cli/src/fonts.rs#L36-L55), [current selection](https://github.com/typst/typst/blob/89495172a5b6a697cbf2d6c560efb2da9dfd173b/crates/typst-library/src/text/font/book.rs#L139-L163))
3. **Dependency-set guarantee:** upstream names CLI output as the current compilation's dependencies, but the compiler interface has no dependency contract and the implementation is an unordered attempted-access log with known non-file omissions. The primary sources do not specify whether consumers may rely on it as a complete reproducible build closure. ([CLI wording](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/args.rs#L360-L376), [`World` boundary](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-library/src/lib.rs#L44-L98), [access-log semantics](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/files.rs#L81-L116))
4. **Symlink watch invalidation:** upstream passes the realized, possibly symlink-containing `PathBuf` to `notify::RecommendedWatcher`, but Typst does not specify cross-platform behavior for replacing the link, modifying its target, or retargeting an ancestor link. The Typst source therefore does not establish a portable invalidation rule for these cases. ([resolved dependency path](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L246-L269), [watch registration](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/watcher.rs#L87-L107))
5. **Package inventory invalidation:** upstream defines how an existing package is searched and downloaded, but watch tracks requested files only after a package root resolves. No first-party contract says which local directory, registry index, cache mutation, or network event should invalidate a previously unavailable or newly selected package. ([package lookup](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-kit/src/packages.rs#L84-L141), [dependency filter](https://github.com/typst/typst/blob/3ae52774b48987fc78a72ff483068cacc28e46c2/crates/typst-cli/src/world.rs#L97-L101))
